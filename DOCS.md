# Documentation

## Architecture

App Asset Generator is a single-page application with no server-side logic. The entire app consists of three files:

| File | Purpose |
|------|---------|
| `index.html` | Page structure: upload area, asset checkboxes, color/scale controls, generate button, preview grid |
| `style.css` | Visual styling with CSS variables, responsive layout, animations |
| `app.js` | All logic: file handling, image processing, ZIP packaging, download |

### Data Flow

```
Source Image (drag & drop or file picker)
    ↓
HTMLImageElement stored in memory
    ↓
User selects asset types + configures colors/scale/threshold
    ↓
Canvas-based resize/transform for each selected asset
    ↓
JSZip collects all generated blobs + metadata files (Contents.json, XML)
    ↓
FileSaver.js triggers ZIP download
```

## How Image Processing Works

All transformations use the HTML Canvas API with `imageSmoothingQuality: 'high'`.

### Icon Resizing (`resizeToSquare`)

Draws the source image scaled to the target square size. Used for iOS icons, legacy Android launchers, favicons, and store icons.

### Round Icons (`resizeToRound`)

Same as square resize but applies a circular clip path before drawing. Used for Android `ic_launcher_round.png`.

### Adaptive Icon Foreground (`createAdaptiveForeground`)

Android adaptive icons use a 108dp canvas where the system masks the outer edges. The visible area is 72dp centered in the middle. This function draws the source image into that 72dp zone so nothing gets clipped by the launcher's mask shape.

### Splash Screens (`createSplash`)

Fills the canvas with the selected background color, then draws the source image centered at the configured scale percentage. Preserves the source image's aspect ratio.

### Monochrome Icons (`createMonochrome`)

Converts the source image to a white silhouette on a transparent background for Android 13+ themed icons:

1. Draw the source image at 108dp (full adaptive icon layer)
2. For each pixel, compute luminance: `0.299*R + 0.587*G + 0.114*B`
3. If luminance is below the threshold (or above, if inverted) and alpha > 30, set to white; otherwise, set to transparent
4. The Android system tints this white silhouette to match the user's wallpaper

### Notification Icons (`createNotificationIcon`)

Similar to monochrome but at 24dp with 2dp padding. Produces a white silhouette on transparent background for the Android status bar and notification drawer.

## Generated Metadata Files

### iOS `Contents.json` (Icons)

Xcode-compatible manifest mapping each icon PNG to its idiom (iphone/ipad/ios-marketing), scale, and point size. Generated by `buildContentsJson()`.

### iOS `Contents.json` (Launch Images) — Legacy

> **Deprecated:** `LaunchImage.launchimage` is deprecated since iOS 13 (2019). Apple requires `LaunchScreen.storyboard` for all new apps.

Maps each splash screen PNG to its device class, orientation, scale, minimum system version, and screen height subtype. Generated by `buildLaunchImageContentsJson()`.

### Android `ic_launcher.xml` / `ic_launcher_round.xml`

Adaptive icon XML referencing the background drawable (solid color), foreground layer, and optionally the monochrome layer. Placed in `mipmap-anydpi-v26/`.

### Android `ic_launcher_background.xml`

A `<shape>` drawable with a solid fill matching the configured icon background color.

### Android `colors.xml`

Two versions:
- `values/colors.xml` — light theme colors (icon background + splash background)
- `values-night/colors.xml` — dark theme colors

## Integrating Generated Assets

### iOS (Xcode)

1. Unzip and locate the `ios/AppIcon.appiconset/` folder
2. In Xcode, open `Assets.xcassets`
3. Delete the existing `AppIcon` set
4. Drag the `AppIcon.appiconset` folder into `Assets.xcassets`
5. For splash screens, drag `LaunchImage.launchimage` into `Assets.xcassets` and configure your launch storyboard or `Info.plist` accordingly

### Android (Android Studio)

1. Unzip and locate the `android/res/` folder
2. Copy the contents into your project's `app/src/main/res/` directory, merging with existing folders
3. The adaptive icon XML in `mipmap-anydpi-v26/` will be picked up automatically for API 26+
4. For dark theme support, `values-night/colors.xml` provides the dark variant of icon and splash colors

### Web

1. Copy files from the `web/` folder to your site's root
2. Add to your HTML `<head>`:

```html
<link rel="icon" href="/favicon.ico" sizes="48x48">
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
<link rel="manifest" href="/site.webmanifest">
```

The `site.webmanifest` references `android-chrome-*` and `maskable-icon-*` files for PWA install prompts. Edit the `name` and `short_name` fields to match your app.

## Customization

### Adding New Icon Sizes

Edit the corresponding array in `app.js`:

- `IOS_ICONS` — add entries with `name`, `size`, `idiom`, `scale`, `sizeStr`
- `ANDROID_DENSITIES` — add entries with `folder`, `size`, `adaptiveLayer`
- `ANDROID_NOTIFICATION` — add entries with `folder`, `size`
- `FAVICON_SIZES` — add entries with `name`, `size`, and optionally `maskable: true` for PWA maskable icons

### Adding New Splash Screen Sizes

- `IOS_SPLASH` — add entries with `name`, `w`, `h`, `orientation`, `idiom`, `scale`, `minVer`, and optionally `subtype`
- `ANDROID_SPLASH` — add entries with `folder`, `w`, `h`

The `Contents.json` and ZIP structure update automatically based on these arrays.

### Changing the Monochrome Algorithm

The `createMonochrome()` function in `app.js` controls how the silhouette is generated. You can modify:
- The luminance formula (currently ITU-R BT.601)
- The alpha threshold for foreground detection (currently `a > 30`)
- The output color (currently white `#FFFFFF`)

## Dependencies

Both loaded from CDN in `index.html`:

| Library | Version | Purpose |
|---------|---------|---------|
| [JSZip](https://stuk.github.io/jszip/) | 3.10.1 | Generates ZIP archives in the browser |
| [FileSaver.js](https://github.com/nicolo-ribaudo/FileSaver.js) | 2.0.5 | Triggers file download from a Blob |

No npm, no bundler, no build step.
